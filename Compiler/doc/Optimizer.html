    optimizer_bitwiseN(ast_nodo, left, right)

    Optimiza el bitwise negation en dado caso que sea posible. Reeplaza el nodo dado la siguiente regla: Si hay una expresion ~~5 regresará 5 y asi de manera ascendente en el árbol. ##Parámetros -ast_nodo : Se refiere al nodo actual -left : Hijo izquierdo del nodo actual optimizado o no. -right : Hijo derecho del nodo actual optimizado o no.

    optimizer_log_neg(ast_nodo, left, right)

    Optimiza la negacion lógica de tal manera que si encuentra tres negaciones seguidas se reducirá a una. Por ejemplo !!!5, esto primero dará 0, luego 1 y finalmente 0 por lo que es mas optimo solo utilizar una vez el !. Asimismo con numeros paresde negacion como: !!!!5 lo cual dara como resultado !!5. Si se cumple la condicion regresará el la expresion antes mencionada. Si no, entonces regresará un nodo con sus hijos left y right optimizados si asi sea. ##Parámetros -ast_nodo : Se refiere al nodo actual -left : Hijo izquierdo del nodo actual optimizado o no. -right : Hijo derecho del nodo actual optimizado o no.

    optimizer_neg(ast_nodo, left, right)

    Optimiza la negacion en dado caso que sea posible. Reeplaza el nodo dado la siguiente regla: Si hay una expresion --5 regresará 5 y asi de manera ascendente en el árbol. ##Parámetros -ast_nodo : Se refiere al nodo actual -left : Hijo izquierdo del nodo actual optimizado o no. -right : Hijo derecho del nodo actual optimizado o no.
